PROJECT 01

NYT News Analyzer

Python Programming · Final Exam Project

Overview

A fully interactive Python application that queries the New York Times Article Search API to analyze keyword trends and publishing patterns across news sections. Users define the section, date range, and time granularity — the app handles data collection, analysis, and visualization end-to-end.

Key Features

  • Built a dynamic API integration using requests to fetch and paginate NYT articles by section, keyword filter, and date range — with the API key stored securely in a separate config file
  • Used pandas to extract, flatten, and structure article data (headlines, publication dates, keyword arrays) into an analysis-ready DataFrame
  • Computed keyword frequency rankings to surface the most-discussed topics within any given section and time window
  • Implemented flexible time-series resampling (daily / weekly / monthly) to track how article volume and topic interest shift over time
  • Generated three distinct visualizations: a horizontal bar chart of top keywords, a line graph of publishing cadence over time, and a publication-date bar chart of top headlines

Visualizations Produced

  • Horizontal bar chart — top 10 most frequently mentioned keywords in the selected section
  • Line graph — article volume over time at user-selected periodicity (daily / weekly / monthly)
  • Headline bar chart — top 10 headlines mapped to their publication dates

🔍 Demonstrated live during an oral exam — ran the application end-to-end with live API calls, fielded instructor questions on design decisions and code logic.

PROJECT 02

Personalized Medicine & Patient Satisfaction Dashboard

CIS9655 Data Visualization · Group Final Project · Dec 2025

Overview

A group research project investigating whether the growth of personalized medicine (PM) correlates with increased patient satisfaction measurement. Using 6,023 clinical trials (ClinicalTrials.gov, 2000–2024) and 34,815 hospital observations (CMS HCAHPS data), the team built an interactive six-visualization Python dashboard — and rigorously reported a null result.

Research Question

Does the implementation of personalized medicine approaches affect patient satisfaction and health outcomes across different geographical regions?

Key Findings

  • Hypothesis NOT supported: Statistical analysis found no significant relationship between PM adoption and patient satisfaction measurement (r = −0.20, R² = 0.04, p = 0.40)
  • PM growth was dramatic: Personalized medicine trials grew 2,500%+ from near-zero in 2000 to 520 trials in 2024 — a textbook “hockey stick” curve
  • Satisfaction trended opposite: The highest PM era (2020–2024) showed lower satisfaction measurement (~28%) than the early 2000s peak (60%), suggesting the two are driven by entirely separate institutional forces
  • Geographic disparities persist: A 1.8-star gap exists between top states (Wisconsin 3.89★, South Dakota 3.88★) and bottom (Puerto Rico 2.07★, DC 2.32★) — Midwest consistently outperforms coastal urban centers

Dashboard Visualizations (6 Interactive Charts)

  • Dual-axis time-series line chart — PM trials vs. traditional trials vs. % measuring satisfaction (2000–2024)
  • US choropleth map — HCAHPS star ratings by state with RdYlGn color scale
  • Scatterplot with regression line — PM adoption % vs. satisfaction measurement % with Pearson r annotation
  • Regional bar chart — top 15 states ranked by patient satisfaction score
  • Enrollment trend chart — total and average patient enrollment over time
  • Study status donut chart — 39.9% completed, 24.9% recruiting, 5% terminated

📊 Datasets: 6,023 clinical trials (ClinicalTrials.gov API v2) + 34,815 hospital observations (CMS Hospital Compare) · Final sample: 3,176 unique hospitals across 51 states

PROJECT 03

Predicting Student Academic Performance: A Data Mining Approach

CIS9660 Data Mining · Group Final Project · Dec 2025

Overview

A machine learning study using the UCI Student Performance dataset (395 students, 33 variables) to build an early warning system for at-risk students. The team developed both regression models (predicting final grades on a 0–20 scale) and classification models (flagging at-risk students before they fail), comparing linear, tree-based, and ensemble approaches with 10-fold cross-validation throughout.

Research Questions

  • Regression: What factors most significantly predict a student’s final grade (continuous 0–20)?
  • Classification: Can we accurately identify at-risk students (grade < 10) early enough for effective intervention?

Model Architecture

To address multicollinearity between G1/G2 prior grades (r = 0.852), three model variants were designed for different intervention windows:

  • Model A — Mid-Semester: Uses G2 (2nd period grade); best overall performance
  • Model B — Early-Semester: Uses G1 (1st period grade) only
  • Model C — Pre-Semester: No prior grades; demographics & behavioral factors only

Key Results

  • Best regression model: Random Forest (Model A) — R² = 0.872, RMSE = 1.58 grade points, explaining 87% of variance in final grades
  • Best classification model: Random Forest (Model A) — F1 = 0.871, Recall = 89.8%, ROC-AUC = 0.951; only 2 at-risk students missed in the test set
  • Dominant predictor: G2 (prior grade) accounts for 84.3% of Random Forest feature importance — academic trajectories are highly stable within a semester
  • Critical modifiable factor: Absences rank 2nd at 12.5% importance — the only non-grade predictor exceeding 1%, and directly actionable by schools
  • Early intervention ceiling: Without any prior grades (Model C), best R² drops to 0.29, revealing the fundamental limits of demographic-only prediction

🏫 Practical output: A 4-stage early warning system for administrators — from Week 1 demographic screening through Week 10 intensive intervention — with specific model recommendations for each stage and resource-constrained settings.

Algorithms Compared

  • Linear Regression & Ridge Regression (baseline + regularized)
  • Logistic Regression (classification baseline)
  • Decision Tree Regressor & Classifier (max_depth = 5)
  • Random Forest Regressor & Classifier (100 trees, max_depth = 10)
